# Supabase Setup

Slik uses Supabase for authentication and database. This guide walks you through the complete setup.

## 1. Create a Supabase Project

1. Go to [supabase.com](https://supabase.com)
2. Click **"New project"**
3. Choose your organization
4. Enter project name, database password, and region
5. Click **"Create new project"** and wait ~2 minutes

## 2. Run the Database Migration

In your Supabase project:
1. Go to **SQL Editor** (left sidebar)
2. Click **"New query"**
3. Copy the contents of `supabase/migrations/001_initial.sql` from your project
4. Paste and click **"Run"**

This creates:
- `profiles` table with RLS policies
- Auto-create profile trigger on signup
- Admin role support

## 3. Enable Google OAuth

1. Go to **Authentication → Providers**
2. Find **Google** and toggle it on
3. Add your Google Client ID and Secret
4. Get these from [Google Cloud Console](https://console.cloud.google.com)

## 4. Configure Redirect URLs

In Supabase: **Authentication → URL Configuration**

Add to **Redirect URLs**:
http://localhost:3000/api/auth/callback
https://your-domain.com/api/auth/callback

## 5. Get Your API Keys

Go to **Project Settings → API**:

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGc...
```

<Warning>
  Never expose SUPABASE_SERVICE_ROLE_KEY in client-side code. It's for server-side only.
</Warning>
